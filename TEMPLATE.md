# 여행 일정 웹앱 — 다른 지역 재사용 가이드 (템플릿 & 프롬프트)

이 폴더(`포르투갈 여행`)는 **모바일 친화 반응형 여행 일정 PWA**의 한 인스턴스입니다.
이 문서는 **똑같은 기능을 다른 여행지(예: 일본·이탈리아·스페인…)에 그대로 복제**할 때
무엇을 바꾸면 되는지, 그리고 **AI에게 그대로 붙여넣어 새 버전을 만들게 하는 프롬프트**를 담고 있어요.

> 핵심 원칙: **로직(JS 함수)·CSS·PWA 골격은 그대로 두고, "데이터 블록"만 갈아끼우면 된다.**
> 모든 콘텐츠는 `index.html` 안의 몇 개 `const` 배열/객체에 들어있습니다.

---

## 1. 아키텍처 한눈에

순수 정적 사이트 — 빌드 도구·프레임워크·백엔드 없음. GitHub Pages 등 정적 호스팅에 그대로 올림.

```
index.html        HTML + CSS + JS 전부 한 파일 (콘텐츠 데이터도 여기 안의 const들)
manifest.json     PWA 설치 정보(이름·색·아이콘)
sw.js             오프라인 캐시 서비스워커 (CACHE 버전 문자열 보유)
fonts/            나눔스퀘어 woff2 (R/B/EB) — 셀프호스팅, 오프라인 OK
icon-192.png      앱 아이콘
icon-512.png      앱 아이콘
```

**외부 의존(런타임, 키 불필요):**
- 날씨: Open-Meteo API (`api.open-meteo.com`) — 위경도만 있으면 됨, 무료·키 없음
- 사진: 위키백과(Wikipedia REST + `upload.wikimedia.org`) — 장소명(영문)으로 자동
- 지도/길찾기: Google Maps URL 스킴
- 후기: 네이버 블로그 검색 URL
- 투어 예약: GetYourGuide / Viator / Klook / 마이리얼트립 검색 URL

이 의존들은 **지역과 무관하게 그대로 동작**합니다(위경도·검색어만 바뀜).

## 2. 핵심 동작(그대로 유지되는 기능들)

| 기능 | 구현 위치(함수/요소) | 지역 의존성 |
|---|---|---|
| D-Day 카운터 | `updateDday()` | 여행 시작·종료 **날짜**만 |
| 두 시간대 실시간 시계 | `updateClocks()` / `tzNow()` | 현지 **타임존 문자열** (`Europe/Lisbon`) |
| 도시별 날씨 | `WXCITY`, `loadWeather()` | 도시 **위경도 + 해당 날짜** |
| €→₩ 환율 변환기 | `RKEY`, `rateEl` | 통화 기호·기본 환율 |
| 일정(펼치기/편집/오늘 강조) | `DAYS`, `render()` 계열 | **DAYS 데이터** |
| 하루 동선(구글맵) | `STAYS`, `LEGS`, `routeBtn()` | 숙소·구간 데이터 |
| 장소 상세 페이지(해시 라우팅) | `PLACE_INFO`, `STORY`, 위키 fetch | 장소 설명 데이터 |
| 근처 먹거리 | `FOODS` | 식당 데이터 |
| 추천 1일 투어 | `TOURS`, `TOURP` | 투어 데이터 |
| 예약 체크리스트 진행률 | `items[].res`, `CKEY` | DAYS 안의 `res:1` 플래그 |
| 준비물 체크리스트 | `PACK`, `PKEY` | 준비물 데이터 |
| 경비 가계부·정산 | `costs`, `ECATS`, `EKEY` | 비용 항목·인물명 |
| 현지어 회화 | `PHRASES` | 언어 데이터 |
| 다크모드 / 인쇄·PDF | CSS + 토글 | 없음 |
| PWA 설치·오프라인 | `manifest.json`, `sw.js` | 이름·색·캐시버전 |

## 3. 갈아끼울 "데이터 블록" 목록 (전부 `index.html` 안)

각 `const`의 스키마와 의미. **함수는 건드리지 말고 데이터만 교체**하세요.

### 3.1 `DAYS` — 일정의 본체 (가장 중요)
하루 = 객체 하나. `items` 배열이 그 날의 일정/장소.

```js
const DAYS=[
 { date:"7/1", dow:"수", title:"리스본 도착", wake:"—", items:[
    { t:"20:15",                // 시간 또는 "오전/점심/일몰/저녁" 같은 라벨
      name:"리스본 공항 도착 (Lisbon Airport)",  // 표시 이름(현지어 병기 권장)
      s:5,                       // ★별점/추천도 1~5 (선택) — 장소 강조용
      res:1,                     // 예약 필요 항목이면 1 → 예약 체크리스트에 자동 포함(선택)
      liq:1,                     // 주류/와인 구매처 표시 플래그(선택)
      desc:"KE921 도착 → 입국·수하물",  // 짧은 설명
      tip:"공항~시내 택시 15~25분",     // 팁(선택)
      m:"Lisbon Airport",        // ★구글맵/위키 사진용 "영문 검색어"(장소 상세의 KEY) — 영문 권장
      n:"리스본 공항",            // 네이버 후기 검색어(한국어, 선택)
      b:"https://... 또는 검색어" // 예약 링크 또는 검색어(선택). http로 시작하면 링크, 아니면 구글검색
    },
 ]},
];
```
- `m`(영문 장소명)은 **장소 상세 페이지의 고유 키**이자 위키 사진 검색어예요. 빠지면 그 항목은 상세/사진이 안 떠요.
- 날짜는 `"월/일"` 문자열. 연도는 `isoOf()`/`updateDday()`에서 **하드코딩(2026)** 되어 있으니 §4 참고해 바꾸세요.

### 3.2 `FOODS` — 날짜별 근처 먹거리
```js
const FOODS={
 "7/1":[ { name:"봉자르딤 ...", desc:"...", m:"Bonjardim Lisbon",
           w:"Piri piri",   // 위키 "음식" 영문명(사진용, 선택)
           n:"리스본 봉자르딤" } ],   // 네이버 검색어(선택)
};
```
키 = `DAYS`의 `date`와 동일. 해당 날 카드 하단에 "이 근처 먹거리"로 표시.

### 3.3 `TOURS` — 날짜별 추천 1일 투어 (+ `TOURP` 예약 플랫폼)
```js
const TOURS={
 "7/2":[ { name:"신트라 투어", price:"€50~", desc:"...",
           q:"Sintra day tour",      // 글로벌 플랫폼 검색어(영문)
           kq:"신트라 투어",          // 마이리얼트립 등 한국어 검색어
           rec:{c:"red", t:"강력추천 ", why:"렌터카보다 투어가 편함"} } ],
};
```
`rec.c`는 `red`(강력추천)/`blue`(개인추천)/`""`(선택). `TOURP`(플랫폼 목록)는 지역 무관하게 그대로 둬도 됨.

### 3.4 `STAYS` — 숙소(동선 출발지)
```js
const STAYS=[
 { from:"7/1", to:"7/3", mapq:"호텔 구글맵 검색어",
   label:"더7호텔", mode:"transit",  // walking|transit|driving 기본 추천
   decided:true },                    // 확정 여부(false면 "미정" 뱃지)
];
```

### 3.5 `LEGS` — 날짜별 구간 이동(이모지 + 소요시간 텍스트)
```js
const LEGS={
 "7/3":[ {m:"🚊", t:"호텔 → 벨렝 · 트램15E 약 25분"}, ... ],
};
```
순수 표시용. `routeBtn()`이 `STAYS`+`DAYS[].items[].m`로 구글맵 길찾기 버튼을 자동 생성.

### 3.6 `PLACE_INFO` & `STORY` — 장소 상세 콘텐츠
```js
const PLACE_INFO={
 "Jerónimos Monastery Lisbon":{        // 키 = DAYS의 m 값과 정확히 일치
    wk:"Jerónimos Monastery",          // 위키백과 영문 제목(사진·요약, 선택)
    kw:"제로니무스 수도원",             // (선택) 한국어 별칭
    ko:"한국어 설명 한두 문장." },
};
const STORY={ "Praça do Comércio Lisbon":"더 긴 비하인드 스토리 한 단락..." };
```
키가 `DAYS[].items[].m`(또는 `FOODS[].m`)과 **글자까지 일치**해야 연결됩니다.

### 3.7 `WXCITY` — 날씨 카드
```js
const WXCITY=[
 { name:"리스본", lat:38.72, lon:-9.14,
   dates:["2026-07-01","2026-07-02"],   // 이 도시에 머무는 날(YYYY-MM-DD)
   clim:"낮 <b>28°</b> / 밤 17° · 맑고 건조" },  // 평년 기후 한 줄
];
```
실제 예보는 출발 ~2주 전부터 Open-Meteo로 자동 표시(위경도만 맞으면 됨).

### 3.8 `costs` — 예상 비용(2인 기준)
```js
const costs=[ {l:"✈ 국제선", e:2600, s:"부연설명"}, ... ];  // e=유로 금액
```
`renderCost()`가 13일/2인으로 나눔 → **여행 일수·인원이 다르면 함수의 `/13`, `/2`도 확인**(§4).

### 3.9 `PACK` — 준비물 체크리스트
```js
const PACK=[ {g:"필수 서류·결제", items:["여권","유로 현금", ...]}, ... ];
```
일정 맞춤 항목(아쿠아슈즈·방한 등)은 새 지역 활동에 맞게 갈아끼우세요.

### 3.10 `PHRASES` — 현지어 회화
```js
const PHRASES=[ {g:"기본·인사",
   items:[ ["감사합니다","Obrigado","오브리가두"], ... ]}, ];  // [한국어, 현지어, 한글발음]
```
새 지역 언어(일본어·이탈리아어 등)로 교체.

### 3.11 파두(FADO) 같은 지역 특화 블록
포르투갈판엔 `DAYS` 위에 **파두 공연장 배열**이 있어요(현지 문화 콘텐츠). 다른 지역에선
이런 특화 블록을 **그 지역 것으로 바꾸거나 삭제**하면 됩니다(예: 일본=온천/이자카야, 이탈리아=오페라).

## 4. 지역과 무관하게 "코드에 하드코딩된" 곳 (꼭 함께 수정)

`index.html`에서 검색해 바꾸세요.

| 찾을 것 | 무엇 | 새 값 예시 |
|---|---|---|
| `"2026-"` (in `isoOf`) | 일정 연도 prefix | 여행 연도 |
| `new Date(2026,6,1)` / `new Date(2026,6,13)` (in `updateDday`) | 시작·종료일(월은 0-기반: 6=7월) | 새 여행 시작/끝 |
| `tzNow("Europe/Lisbon")` (in `updateClocks`) | 현지 타임존 | 예: `Asia/Tokyo` |
| `🇵🇹 포르투갈` / `🇰🇷 한국` 라벨 | 시계 라벨·국기 | 새 국기·국가명 |
| `/13`, `/2` (in `renderCost`) | 일수·인원 분모 | 새 일수·인원 |
| 통화 기호 `€`, 환율 기본값(`RKEY` 근처) | 통화 | 예: `¥`, `₩`/100엔 |
| `revUrl`(네이버) | 후기 검색 — 그대로 둬도 OK | — |
| 결제 인물명 `은혜`/`창민` (경비 정산) | 동행자 이름 | 새 이름 |

**localStorage 키**(접두사 `pt_`): `pt_itin_v1`, `pt_resv_v1`, `pt_rate_v1`, `pt_wiki_v1`, `pt_pack_v1`, `pt_exp_v1`, `pt_wx_v1`.
다른 지역으로 만들 땐 충돌·혼동 방지를 위해 접두사를 `jp_`/`it_` 등으로 일괄 변경 권장
(상수 `KEY/CKEY/RKEY/WKEY/PKEY/EKEY` + `pt_wx_v1` 문자열).

**PWA·메타데이터:**
- `manifest.json`: `name`/`short_name`/`description`/`theme_color`/`background_color`/`lang`
- `index.html` `<title>`, `<meta name="theme-color">`, 히어로 제목/`eyebrow`(`Portugal · 13 Days`)
- `sw.js`: `CACHE` 문자열(예: `jp-trip-v1`)을 **반드시 새 값으로** (캐시 강제 갱신)
- `icon-192/512.png`: 새 아이콘으로 교체(선택)

## 5. 새 지역 만들기 — 절차 요약

1. 이 폴더 전체를 새 폴더로 복사 (`fonts/`, 아이콘 포함 / `nanum-all_new*`는 제외 가능 — `.gitignore` 참고)
2. `index.html`의 §3 데이터 블록을 새 지역 콘텐츠로 교체
3. §4 하드코딩 값(연도·날짜·타임존·통화·인원·localStorage 키) 수정
4. `manifest.json` / `<title>` / `theme_color` / `sw.js`의 `CACHE` 갱신
5. 로컬에서 확인: 정적 서버로 열기
   ```bash
   python -m http.server 8000   # http://localhost:8000
   ```
   (서비스워커·fetch는 `file://`에서 제한 → 반드시 http로 확인)
6. 배포:
   ```bash
   git add . && git commit -m "새 지역 일정" && git push
   ```
   GitHub Pages면 1~2분 뒤 반영. SW 캐시 갱신은 `sw.js`의 `CACHE` 버전을 올리면 강제.

## 6. 그대로 복붙해서 쓰는 AI 프롬프트

아래를 새 채팅(또는 새 폴더에서 Claude Code 등)에 붙여넣고 **대괄호 부분만 채우세요.**
이 `TEMPLATE.md`와 기존 `index.html`을 함께 첨부/참조시키면 가장 정확합니다.

```text
첨부한 여행 일정 PWA(index.html + manifest.json + sw.js + fonts/)를
"[새 여행지: 예) 일본 간사이]" 여행용으로 그대로 복제해줘. 기능·디자인·레이아웃·
PWA·오프라인·해시 라우팅 장소 상세·날씨·환율·경비·준비물·회화 모듈은 100% 유지하고,
TEMPLATE.md의 §3 데이터 블록과 §4 하드코딩 값만 새 지역에 맞게 교체해.

[여행 정보]
- 여행지/도시들: [예) 오사카·교토·나라·고베]
- 기간: [예) 2027-04-01 ~ 2027-04-09 (8박 9일)]
- 인원/동행자 이름: [예) 2인 · 민수·지영]
- 통화/기본환율: [예) 100엔 = 약 900원, 기호 ¥]
- 현지 타임존: [예) Asia/Tokyo], 시계 라벨/국기: [🇯🇵 일본 / 🇰🇷 한국]
- 도시별 위경도(날씨용): [예) 오사카 34.69,135.50 / 교토 35.01,135.77 ...]
- 항공/숙소/예약 확정 정보: [아는 만큼 — 없으면 추천으로 채워줘]
- 현지 특화 콘텐츠: [예) 파두 블록 대신 '온천/이자카야 추천' 블록]
- 회화 언어: [예) 일본어 — 한국어/현지어/한글발음 3열]

[요구사항]
1) DAYS: 날짜별 일정. 각 장소엔 m(영문 구글맵/위키 검색어)를 꼭 넣어 장소 상세·사진이 뜨게.
   예약 필요 항목은 res:1, 추천도는 s(1~5), 주류 구매처는 liq:1.
2) FOODS/TOURS/STAYS/LEGS/PLACE_INFO/STORY/WXCITY/costs/PACK/PHRASES 전부 새 지역으로.
   PLACE_INFO/STORY/FOODS의 키는 DAYS의 m 값과 글자까지 일치시킬 것.
3) §4 하드코딩 값 모두 수정: 연도 prefix, updateDday의 시작/종료 Date(월은 0-기반),
   tzNow의 타임존, 시계 라벨, renderCost의 일수/인원 분모, 통화 기호, 동행자 이름.
4) localStorage 접두사 pt_ → [새 접두사 예: jp_] 로 일괄 변경(KEY/CKEY/RKEY/WKEY/PKEY/EKEY + pt_wx_v1).
5) manifest.json(name/short_name/description/색), <title>, meta theme-color, sw.js의 CACHE 문자열,
   히어로 제목·eyebrow 모두 새 지역으로.
6) 함수 로직·CSS·PWA 골격은 변경 금지. 결과는 동작하는 완성된 파일들로.
```
